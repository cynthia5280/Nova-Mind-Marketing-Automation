NovaMind

From blog brief to CRM campaign
in five automated stages.

A demonstration of how NovaMind eliminates the manual work between a content idea and a persona-targeted newsletter campaign — outline, draft, segmentation, HubSpot distribution, and reporting, all automated.

Most agencies write a blog post, then manually adapt it for newsletters, then manually paste contacts into their CRM, then manually pull together a performance summary. That's four hours of copy-paste work for every piece of content.

This pipeline does all of it. You type in a topic, a target audience, and a content goal. Five stages run in sequence. You get a blog draft, three persona-targeted newsletters, HubSpot contacts with campaign notes attached, simulated performance metrics, an AI analysis, and a saved report — without touching anything between step one and the end.

The Pipeline
01 Input Collection main.py · terminal prompt
You put in
  • A blog topic in plain English
  • A description of who will read it
  • What you want the content to achieve
You get out
  • Three structured variables passed to every downstream stage
  • A printed summary confirming the inputs before generation begins
Blog topic: ai automation Target audience: social media agency Content intent: subscription
02 Content Generation Claude Haiku · outline_agent.py + writer_agent.py
You put in
  • Topic, audience, and intent from Stage 1
  • A system prompt that positions the writer as a NovaMind consultant, not a tech blogger
You get out
  • A 5-section blog outline with headline and one-sentence summaries
  • A 400–600 word blog draft built from that outline — specific, numbered, no corporate language

03 Persona Segmentation Claude Haiku · newsletter_agent.py
You put in
  • The blog draft from Stage 2
  • Three separate system prompts — one per audience persona — each with a different angle, vocabulary, and CTA
You get out
  • Three distinct 150–200 word newsletter versions, each written for a different reader inside the same agency
  • Same core message, completely different tone and hook for each persona

04 CRM Distribution HubSpot API v3 · hubspot_agent.py
You put in
  • Blog topic and all three newsletter versions
  • Three mock contacts (one per persona) defined in the agent
  • A HubSpot access token from the environment
You get out
  • Three contacts created in HubSpot, each tagged with a persona property
  • A campaign note attached to each contact with the newsletter content and send date
  • A summary of contact IDs and note IDs — existing contacts are fetched gracefully on 409 conflict
05 Reporting Claude Haiku · report_agent.py + html_report.py
You put in
  • Topic, all three newsletter versions, and the HubSpot summary
  • No manual data entry — everything is passed automatically from prior stages
You get out
  • Simulated open rate, click rate, and unsubscribe rate per persona
  • An AI-written analysis comparing performance across the three audiences with one specific recommendation for next week
  • A JSON run log and a standalone HTML report saved to outputs/

AI Analysis · Claude Haiku

The PM persona significantly outperforms both segments on click-through rate (7.1% vs. 4.0% and 3.6%), despite comparable open rates. This audience resonates strongly with the approval and workflow friction angle — the chaos of managing 50+ clients through Slack and spreadsheets is a vivid, specific pain point that lands.

Recommendation for next week: Shift 60% of sends to the PM persona and test a subject line that emphasises time saved per week rather than annual cost — e.g. "Save Your Team 6 Hours Every Wednesday." The workflow-specific pain point is driving engagement, and quantifying weekly relief may push CTR higher still.

Output

What gets saved after every run.

Two files written to outputs/ automatically. No manual export step.

outputs/run_{timestamp}.json
Full Run Log
Every input and output from the pipeline serialised to JSON. Machine-readable, easy to load into a dashboard or pass to a downstream process.
  • Run timestamp, topic, audience, intent
  • Full blog outline and draft
  • All three newsletter versions
  • HubSpot contact and note IDs
  • Per-persona metrics and AI analysis text
outputs/report_{timestamp}.html
Visual HTML Report
A standalone, self-contained HTML file you can open directly in a browser. No server required. Shares cleanly as an attachment.
  • NovaMind-branded header with topic and run date
  • Three persona metric cards, colour-coded against average
  • Chart.js bar chart comparing click rates across personas
  • Full AI analysis section
  • HubSpot distribution summary with contact and note IDs
